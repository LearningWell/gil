API documentation: stages/<seqNo>/commands/<name>

Invokes the control command with the given name on the transfer pipeline stage with the given seqNo.

URL:

api/pipeline/stages/<seqNo>/commands/<name>[.format]

Formats:

html, xml, json

HTTP Method(s):

POST

Request examples:

Examples using curl tool which is freely downloadable.

html example:

curl -v -X POST -d "doBypass=true" http://localhost:9980/api/pipeline/stages/0/commands/bypass

xml example:

curl -v -X POST --data-binary "<parameters><parameter><name>doBypass</name>
  <value>true</value></parameter></parameters>" -H"Content-Type: application/xml" 
  http://localhost:9980/api/pipeline/stages/0/commands/bypass

json example:

curl -v -X POST --data-binary "{\"parameter\":[{\"name\":\"doBypass\",\"value\":\"true\"}]}" 
  -H"Content-Type: application/json" http://localhost:9980/api/pipeline/stages/0/commands/bypass
