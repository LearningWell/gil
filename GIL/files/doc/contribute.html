Contribute

  1. Fork the GIL source.
  2. Follow the coding style of the already existing source.
  3. Use spaces instead of tabs when indenting.
  4. Supply a new or changed JUnit test case verifying your code.
  5. Supply new html documentation or change the existing if your change needs documentation.
  6. Send a pull request.
